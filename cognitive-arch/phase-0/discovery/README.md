# discovery — pre-project context (placeholder)

BRIEF: Placeholder folder for product-discovery work. If you run a separate "discovery" AI session BEFORE the project (to validate what to build), fill these files. Otherwise, leave as-is.

## Why this exists

The cognitive architecture is for IMPLEMENTATION. But before implementing, you might want to:
- Validate the problem
- Understand the users
- Sketch the solution
- Choose the stack

That's "discovery." It's a separate phase, often done in a different conversation with a different AI focus.

## Files

| File | Purpose |
|------|---------|
| `00-what-and-why.md` | Problem + solution sketch |
| `01-users-and-context.md` | Who uses this and in what context |
| `02-stack-and-scope.md` | Stack rationale and MVP scope |

## When to fill

- Before BOOTSTRAP, if doing structured discovery
- During BOOTSTRAP, abbreviated (just enough to start)
- Skip entirely if discovery is implicit / already settled

## Relationship to PROJECT.md and 00-project-overview.md

These files are upstream of PROJECT.md. They feed the decisions captured in PROJECT.md and Phase 0 files. Discovery is exploration; PROJECT.md is the commitment.

If you do thorough discovery: these files have substance, and PROJECT.md derives from them.

If you skip discovery: leave these as-is; PROJECT.md starts from your existing thinking.
