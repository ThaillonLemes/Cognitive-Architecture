# Template: proposal.md (Governance Proposal)

BRIEF: Schema for proposals generated by sdk/protocol_updater.py when a pattern exceeds threshold. Human reviews, accepts or rejects. Accepted proposals optionally trigger file patches.

Copy to: `governance/proposals/<YYYY-MM-DD>-<pattern_id>.md`

---

```yaml
---
id: proposal-<slug>
status: pending                       # pending | accepted | rejected
pattern_id: <pattern-id>              # e.g. duration-overrun-recurring
target_file: protocols/<file>.md      # the file this proposal suggests changing
proposed_change: |                    # free-text: what change to make (diff, instruction, or text)
  <description of proposed change>
rationale: |                          # why this pattern warrants this change
  <rationale extracted from pattern analysis>
created_at: YYYY-MM-DD
resolved_at: ~                        # null until accepted/rejected
resolved_by: ~                        # agent or human that resolved
signal_count_d1: 3                    # pattern signal count that triggered this proposal
---
```

<proposed change text or diff>

## 5. Resolution

**Status:** pending

_Human: set status to `accepted` or `rejected`, fill `resolved_at` and `resolved_by`._

_To apply: `python sdk/proposal_resolver.py --accept <id> [--apply]`_

---

## Field Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| id | string | ✓ | Unique slug derived from pattern_id |
| status | enum | ✓ | pending \| accepted \| rejected |
| pattern_id | string | ✓ | Pattern that triggered this proposal |
| target_file | string | ✓ | File to patch if accepted |
| proposed_change | text | ✓ | What to change (free text / diff) |
| rationale | text | ✓ | Why this change is warranted |
| created_at | date | ✓ | Auto-set by protocol_updater.py |
| resolved_at | date | ✗ | Null until resolved |
| resolved_by | string | ✗ | Null until resolved |
| signal_count_d1 | int | ✓ | Pattern signal count at creation time |
