# templates — file templates for recurring artifacts

BRIEF: Skeletons for files generated repeatedly during a project. Each template has a paired protocol in `protocols/` that describes HOW to fill it. Copy the template, fill placeholders, save in the destination folder.

## Files in this folder

| Template | Generates | Destination | Paired protocol |
|----------|-----------|-------------|-----------------|
| `phase.md` | Phase doc | `phases/phase-<N>.md` | `protocols/phase-generation.md` |
| `manifest-small.md` | Block manifest (Tier S) | `manifests/block-<NNN>-<slug>.md` | `protocols/manifest-small-generation.md` |
| `manifest-medium.md` | Block manifest (Tier M) | `manifests/block-<NNN>-<slug>.md` | `protocols/manifest-medium-generation.md` |
| `manifest-large.md` | Block manifest (Tier L) | `manifests/block-<NNN>-<slug>.md` | `protocols/manifest-large-generation.md` |

| `phase-retrospective.md` | Phase retro | `phases/phase-<N>-retro.md` | `protocols/phase-retrospective-generation.md` |
| `AGENT.md` | Agent identity | `agents/agent-<name>.md` | `protocols/agents.md` |

## How to use a template

1. Read the matching protocol in `protocols/` (e.g., `phase-generation.md` before using `phase.md`).
2. Copy the template file to the destination path.
3. Fill placeholders (marked `<...>` or `[FILL]`).
4. Save.
5. `audit.sh` (or Governor) validates structure.

## Conventions

- Templates have YAML frontmatter — keep it parseable.
- Sections marked REQUIRED are mandatory; OPTIONAL sections are AI-decided based on context (see paired protocol for rules).
- Don't edit templates themselves to fill a specific instance — copy first.
- If you find a template too rigid OR too loose, propose changes via ADR.
