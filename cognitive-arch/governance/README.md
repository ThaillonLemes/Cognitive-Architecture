# governance — audit reports and overrides

BRIEF: Audit outputs, override dashboard, drift tracking. Generated by Governor or by audit.sh over time.

## What lives here

| Path | Purpose |
|------|---------|
| `audit-<YYYY-MM-DD>.md` | Audit report (generated by `commands/audit.md` or `audit.sh`) |
| `overrides.md` | Dashboard of all axiom overrides used in active blocks |
| `scan-quality-<YYYY-MM-DD>.md` | Quality scan output (`commands/scan-quality.md`) |
| `scan-deps-<YYYY-MM-DD>.md` | Dependency scan output (`commands/scan-deps.md`) |
| `review-<YYYY-MM-DD>.md` | Reviewer agent's findings |
| `doc-drift-<YYYY-MM-DD>.md` | Doc-keeper agent's findings |

## audit-<YYYY-MM-DD>.md

Generated per `templates/audit-report.md`. Run at:
- Every 30 blocks integrated (Governor trigger)
- Every phase close
- Manual request

## overrides.md

A running list of active axiom overrides. Each entry:

```
<block-id> <axiom-id> <YYYY-MM-DD> "<justification>"
```

Audit warns if overrides for the same axiom appear > 5 times in 30 days (drift signal).

## Drift tracking

Periodic audits compare current state to previous audit, tracking:
- Errors / warnings count
- Override frequency
- `forced_pass` blocks count
- Stale lock occurrences
- Pointer breakage trend

Reviewer / doc-keeper agents add their findings here too.

## Retention

Audit reports accumulate. After 6 months, you may archive older ones:
- Move `audit-2026-*.md` → `_archive/audit-2026-*.md`
- Keep last 90 days in main folder

(v1 doesn't automate this — manual hygiene.)
