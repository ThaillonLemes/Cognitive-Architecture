# Cognitive Architecture — AI entry

BRIEF: read first, every session. Routes trigger phrases; sets HOT read order; detects session type.

---

## ⚡ Trigger phrases — auto-route immediately

Full phrase tables + detection markers: `protocols/trigger-phrases.md`. Auto-execute WITHOUT asking.

| Match | Route |
|-------|-------|
| Retrofit phrase | → `RETROFIT.md` (10-step flow, Steps 0–9) |
| Bootstrap phrase | → `BOOTSTRAP.md` (Phase 0 flow) |
| `oi/olá/hello/hi` + workspace empty | → `BOOTSTRAP.md` |
| `oi/olá/hello/hi` + `STATE.md` status:active | → session init + briefing (run `sdk/session_start.py --arch-root .`, read `STATE.md` + `NEXT.md`, reporte status/fase/próxima ação) |
| `oi/olá/hello/hi` + `STATE.md` status:bootstrap ou sem STATE.md + code detected | → ask: retrofit or new project? |

Workspace empty = no `package.json`, `src/`, `app/`, `.git/` (>1 commit). Any match → existing project.

## MANDATORY: Session initializer (run FIRST, before anything else)

```
python sdk/session_start.py --arch-root .
```

This is NOT optional. Run it before reading any files. It:
- Runs all stale tools (health report, pattern mining, dashboard, weekly report, integrity check)
- Updates `last_run` in `governance/tools-registry.yaml`
- Detects session gap and fires briefing if gap > 24h
- Prints phase/status/next_action summary
- Reports active patterns and health score

If the script fails, run each tool manually (see `governance/tools-registry.yaml`).

## HOT read order (every session start — AFTER session_start.py)

1. `PROTOCOLS.md` — 24 axioms (P/Q/C/S) + Charter
2. `STATE.md` — current state (AI-only; see `_syntax.md`)
3. `NEXT.md` — next work pointer
4. `INDEX.md` — file catalog
5. `governance/patterns.md` — active patterns (read this, apply to current work)
6. Health: already generated by session_start.py above — read `governance/health-report-<today>.md`

Memory layers: DX=design/, UX=PROJECT.md, AX=STATE.md+BLOCK_LOG — see `protocols/project-memory-layers.md`.

## Session detection

`STATE.md` has `status:bootstrap` or `PROJECT.md` has unfilled placeholders → run Bootstrap/Retrofit flow.
`STATE.md` has `status:active` → normal work; read `NEXT.md` for pointer.

## Multi-agent

`board.md` rows ≠ `agent:none` → read `agents/agent-<name>.md`; apply mode from `protocols/modes.md`; update board at each state change.
Coordination: `protocols/parallelism.md` · `protocols/agents.md` · `templates/agent-roles/governor.md`
