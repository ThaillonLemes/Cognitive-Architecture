# Command: roadmap-refresh

Mode required: checklist

BRIEF: Re-evaluate the roadmap when design/ has grown or changed significantly. Produces PROPOSALS only — never auto-applies changes. Every proposal requires explicit human confirmation before any phase file is created or modified.

## ⚠️ CRITICAL CONSTRAINT

**This command PROPOSES changes. It NEVER auto-applies them.**

The output of this command is a `_brainstorm/roadmap-refresh-YYYY-MM-DD.md` file containing proposals, each clearly marked as PROPOSAL. The AI must NOT create, modify, or delete any phase file based on this command's output. Human must review and confirm each PROPOSAL before action is taken.

This constraint exists because roadmap changes are high-stakes: a misguided proposal could restructure the entire project plan. Human judgment is required at every step.

---

## When to run

- After adding 2+ new design/ files to an existing project
- After significant domain pivots (change in target users, scope, or technical approach)
- When the user says "the roadmap might be stale" or "I think we're missing phases"
- Periodic health check: every 10–15 completed blocks

---

## Steps

### Step 1 — Audit current roadmap

Run `commands/roadmap-audit.md` in full. Record all errors, warnings, and OK results.

If audit produces 0 errors and 0 warnings: output "Roadmap is healthy — no refresh needed." and stop.

### Step 2 — Identify changed domain concepts

Compare:
- Current `design/` files vs. what was documented when ROADMAP.md was last generated
- Look for: new systems added, existing systems that have new subsystems or changed scope

For each changed concept: note what specifically changed.

### Step 3 — Generate proposals for UNCOVERED gaps

For each UNCOVERED system found in Step 1:

```
PROPOSAL: New phase for [System Name]
  Rationale: System '[System]' is documented in design/[file].md but has no phase or track.
  Suggested action: Create Phase N+X covering [System] using domain-phase-bridge protocol.
  Estimated blocks: S (if simple system), M (if multi-component)
  Prerequisite phases: [phases that must complete first]
  Alternative: Assign to Track [track/name] instead of a phase.
  
  → Human decision needed: create phase, create track, or defer?
```

### Step 4 — Generate proposals for DRIFT phases

For each DRIFT phase found in Step 1:

```
PROPOSAL: Resolve drift in [Phase N]
  Rationale: Phase N references concepts not found in any design/ file.
  Concepts referenced: [list]
  Suggested action A: Document the missing system in design/ (AI can draft if given direction).
  Suggested action B: Merge Phase N into another phase that does have design coverage.
  Suggested action C: Remove Phase N from roadmap (if the work is no longer needed).
  
  → Human decision needed: document system, merge, or remove?
```

### Step 5 — Generate proposals for STALE phases

For each STALE phase found in Step 1:

```
PROPOSAL: Update stale phase [Phase N]
  Rationale: design/[file].md was updated on [date]; Phase N was last revised on [date].
  What changed in design/: [summary of new content]
  Suggested action: Review Phase N exit criteria to incorporate [new subsystem/constraint].
  
  → Human decision needed: update phase scope, add a block, or accept as-is?
```

### Step 6 — Write proposals file

Save all proposals to `_brainstorm/roadmap-refresh-YYYY-MM-DD.md` with:

```markdown
# Roadmap Refresh Proposals — YYYY-MM-DD

## Status: PROPOSALS ONLY — awaiting human review

These are proposals generated by commands/roadmap-refresh.md.
NO phase has been created or modified.
Review each PROPOSAL below and confirm which to act on.

Audit basis: governance/roadmap-audit-YYYY-MM-DD.md
Errors found: N  Warnings found: N

---

[PROPOSAL 1]
[PROPOSAL 2]
...

---

## Instructions for human review

1. For each PROPOSAL marked → Human decision needed: add your decision below the proposal.
2. Tell the AI: "process roadmap-refresh proposals" with your decisions filled in.
3. The AI will then act on confirmed proposals only — no auto-application.
```

### Step 7 — Notify user

Output:
```
Roadmap refresh complete.
Proposals: N new phases, N drift resolutions, N stale updates
File: _brainstorm/roadmap-refresh-YYYY-MM-DD.md

Next step: Review the proposals file and tell me which to act on.
I will NOT make any changes until you confirm.
```
